Fleeing Fame

  • Instructions:
    • Use arrows to move
    • Avoid colliding with enemies/paparazzi
    • Collect the keys by colliding with them
    • Once keys are collected, get to car and drive away
    • After driving away, next level will be loaded or game will end
    • Player should aim to complete all levels (3), minimized number of fails, and minimize time taken

  • Originality: What is original about your game? I think that the style/theme and the way that the game is implemented is unique. The theme is Fame, where the player is the celebrity who must collect his lost keys and drive away before the paparazzi can bother him. This theme drives the game objective and structure, where paparazzi are placed in various positions and traverse in different ways to try to find and take pictures of the player/celebrity. The paparazzi movements are defined by different parametric equations that allow for interesting patterns that can vary in complexity.
  • Technical Merit: Likely the main component of technical merit is found in the use of parametric equations. Parametric equations are used to define the movement patterns of paparazzi, which can vary in complexity. By using parametric equations instead of traditional functions, patterns that defy the vertical line test (no x value can correspond to more than one y value) are allowed. This allowance makes it so shapes such as circles, ellipses, flowers, and stars can be constructed. These equations are implemented by having both x and y correspond to independent equations with respect to a new variable, t. For the paparazzi patterns, I used trigonemetric functions (sin and cos notably) so the movement patterns had some bound, as the functions go from [-1,1] naturally. Research and experimentation was conducted to find patterns the vary in complexity.
  • Prototype Postmortem: What did you learn from this prototype? To accomplish the parametric equations task, I needed some way to evaluate functions. For this task, I found a math library (math.js) which allowed for functions to be defined by a string and evaluated by passing in one or more numbers, so I learned some about that library. I also learned more about parametric equations, notably how they could be use to design certain patterns. What was the easiest or hardest part of making it? The game has multiple levels (3), and I wanted a gradual increase in difficulty between them. One of the components of difficulty that I used was the movement speed of the paparazzi. However, setting the movement speeds proved to be difficult when using parametric equations, due to both the x and y functions being drastically change per each change in t value. The movement speeds are a consequence of how fast these functions change, but with so many different functions it was hard to aim for some standard speed in each level.
